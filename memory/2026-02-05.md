## 关键经验和教训

### Claude Code 使用

**PTY 模式的必要性**
- Claude Code 必须使用 PTY (pseudo-terminal) 模式
- 不使用 PTY 会导致挂起或输出异常
- 在 OpenClaw 中使用 `bash pty:true` 或 Python wrapper

**工作目录的重要性**
- 指定正确的工作目录避免读取无关文件
- 提高上下文质量和性能
- 使用 `workdir:/path/to/project` 参数

**权限管理**
- 使用 `--allowedTools` 遵循最小权限原则
- 限制必要的工具提高安全性

**进度监控**
- 使用 `process:log` 查看后台任务进度
- 使用 `process:poll` 检查是否完成
- 使用 `process:write/submit` 发送输入

### 进度汇报策略

**问题**: 未能每 30 分钟主动汇报进度

### 跨平台兼容性

**macOS 特定问题**
- Python wrapper 使用 `script(1)` 命令在 macOS 上可能不工作
- macOS 上 `script(1)` 的路径或行为与 Linux 不同
