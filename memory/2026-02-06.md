## 任务完成：Python wrapper macOS 兼容性修复

### 问题

Python wrapper (`claude_code_run.py`) 使用 `script(1)` 命令来创建伪终端，但 `script(1)` 在 macOS 上的行为与 Linux 不同，导致：
- Claude Code 挂起
- 输出异常
- 不可预测的行为

### 解决方案

将 `run_with_pty` 函数从使用 `script(1)` 改为使用 Python 标准库的 `pty` 模块：

**修改内容：**
1. 添加 `import pty` 和 `import select`
2. 使用 `pty.openpty()` 创建主从伪终端
3. 使用 `select.select()` 进行非阻塞输出读取
4. 改进错误处理和清理（try/finally）
5. 添加实时输出打印，提升用户体验

**技术改进：**
- 移除了对外部命令 `script(1)` 的依赖
- 跨平台兼容性更好（macOS, Linux, Windows via WSL）
- 更好的输出缓冲处理
- 更健壮的错误处理

### 代码对比

**之前（使用 script(1)）：**
```python
def run_with_pty(cmd: list[str], cwd: str | None) -> int:
    cmd_str = " ".join(shlex.quote(c) for c in cmd)

    script_bin = which("script")
    if not script_bin:
        proc = subprocess.run(cmd, cwd=cwd, text=True)
        return proc.returncode

    proc = subprocess.run([script_bin, "-q", "-c", cmd_str, "/dev/null"], cwd=cwd, text=True)
    return proc.returncode
```

**之后（使用 Python pty）：**
```python
def run_with_pty(cmd: list[str], cwd: str | None) -> int:
    """Run command with a pseudo-terminal for cross-platform compatibility.

    Uses Python's pty module instead of script(1) command for better
    macOS compatibility. script(1) behavior differs between macOS and Linux.
    """
    master_fd, slave_fd = pty.openpty()

    try:
        proc = subprocess.Popen(
            cmd,
            cwd=cwd,
            stdin=slave_fd,
            stdout=slave_fd,
            stderr=slave_fd,
            text=True,
            close_fds=True
        )
        os.close(slave_fd)

        # Read output from the pty
        output = []
        while True:
            r, w, e = select.select([master_fd], [], [], 0.1)
            if master_fd in r:
                try:
                    data = os.read(master_fd, 1024).decode()
                    if data:
                        output.append(data)
                        # Print output in real-time for user visibility
                        print(data, end='')
                except OSError:
                    break

            # Check if process has terminated
            if proc.poll() is not None:
                # Give it a moment to flush remaining output
                time.sleep(0.1)
                # Read any remaining data
                try:
                    remaining = os.read(master_fd, 1024).decode()
                    if remaining:
                        output.append(remaining)
                        print(remaining, end='')
                except OSError:
                    pass
                break

        return proc.returncode
    finally:
        os.close(master_fd)
```

### 关键改进点

1. **跨平台一致性**
   - Python 的 pty 模块在所有平台上行为一致
   - 不依赖外部命令，减少环境差异

2. **更好的错误处理**
   - 使用 try/finally 确保文件描述符正确关闭
   - 使用 `close_fds=True` 防止文件描述符泄漏

3. **实时输出**
   - 使用 `select.select()` 实现非阻塞读取
   - 边读边打印，提升用户体验

4. **更好的资源管理**
   - 及时关闭从端（slave_fd）
   - 主端在 finally 块中确保关闭
